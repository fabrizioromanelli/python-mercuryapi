\subsubsection[{TMR\_\-FirmwareDataProvider}]{\setlength{\rightskip}{0pt plus 5cm}typedef bool($\ast$ {\bf TMR\_\-FirmwareDataProvider})(void $\ast$cookie, uint16\_\-t $\ast$size, uint8\_\-t $\ast$data)}}
\label{tmr__types_8h_d5736b555500b6826cb87096b0d6d783}


Function callback used to obtain firmware data during a firmware load operation. TMR\_\-firmwareLoad will call this repeatedly to obtain data to send to the reader. A buffer of a specified size will be passed in; the callback is free to place less than that amount of data in the buffer, provided that size is modified accordingly. When the end of the firmware image has been reached, the callback should return false.

\begin{Desc}
\item[See also:]\hyperlink{tm__reader_8h_1e57c23bc393527b7e8a51b62c6da4ec}{TMR\_\-memoryProvider} and \hyperlink{tm__reader_8h_ea053ede1e650c1d0b21d95787541e0a}{TMR\_\-fileProvider} for pre-existing callbacks to load firmware from system memory or from a file.\end{Desc}
\begin{Desc}
\item[Parameters:]
\begin{description}
\item[{\em cookie}]Arbitrary data passed to the callback. \item[\mbox{$\leftarrow$} {\em size}]Pointer to amount of space in the buffer. \item[\mbox{$\rightarrow$} {\em Points}]to number of bytes stored in the buffer. \item[\mbox{$\rightarrow$} {\em data}]Buffer of bytes of firmware image. \end{description}
\end{Desc}
\begin{Desc}
\item[Returns:]true if there is more data to load \end{Desc}
